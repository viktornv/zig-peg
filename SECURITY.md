# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it privately instead of opening a public issue.

- Open a private GitHub Security Advisory (preferred).
- Do not report vulnerabilities through public issues.

## What to Include

- Affected version/commit
- Reproduction steps or proof of concept
- Impact and expected severity
- Suggested remediation (if available)

## Disclosure Process

- We aim to acknowledge reports within 3 business days.
- We validate the report and assess impact.
- We prepare a fix and coordinate disclosure timing.
