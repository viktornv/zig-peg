# Support

Use this document to choose the right channel for help.

## Official Channels

- GitHub Issues (primary)
- GitHub Security Advisories (for private vulnerability reports)

## Usage Questions

- Open a GitHub issue with clear reproduction input and expected behavior.
- For parser diagnostics, include grammar snippet, input snippet, and command used.

## Bug Reports

- Use the Bug Report issue template.
- Include Zig version, OS, and test command output.

## Feature Requests

- Use the Feature Request issue template.
- Describe the use case, alternatives, and API/grammar impact.

## Security Reports

- Do not open public issues for vulnerabilities.
- Follow `SECURITY.md` for private reporting.

## Response Expectations

- Responses are provided on a best-effort basis.
- For regular support issues, initial triage is typically within a few business days.
